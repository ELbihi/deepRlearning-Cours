Les briques du RL
Comprendre les concepts fondamentaux avant d'explorer les algorithmes.
Agent
L'entité qui apprend et prend des décisions dans l'environnement.
État S
Représentation de la situation actuelle perçue par l'agent.
Action A
Choix effectué par l'agent à chaque pas de temps.
Récompense R
Signal de feedback sur la qualité de l'action effectuée.

Formule TD de base
V(St) ← V(St) + α × [ Rt+1 + γ·V(St+1) − V(St) ]
L'expression entre crochets est l'Erreur TD = différence entre l'estimation actuelle et la nouvelle estimation bootstrappée.
Q-Learning
Q-Learning
Off-Policy · Model-Free · Value-Based · TD Learning. Le Q signifie « Qualité ».
Off-PolicyModel-FreeValue-BasedTD Learning
Formule de mise à jour
Q(s,a) ← Q(s,a) + α × [ R + γ · maxa' Q(s',a') − Q(s,a) ]
α (alpha)
Taux d'apprentissage
γ (gamma)
Facteur d'actualisation (0 < γ ≤ 1)
max Q(s',a')
Meilleure action théorique dans s'
Politique optimale
π*(s) = argmaxa Q*(s,a)
Les 4 étapes
1
Initialiser la Q-Table
Toutes les valeurs Q(s,a) sont mises à 0 au départ.
2
Choisir action via ε-Greedy
Avec proba ε → aléatoire. Sinon → argmax Q(s,a).
3
Exécuter → observer R et S'
L'agent effectue l'action et observe la récompense et le nouvel état.
4
Mettre à jour Q(s,a)
Appliquer la formule TD avec max_a' Q(s',a').
Comportement — Off-Policy
Q-Learning est audacieux : il optimise pour la meilleure action théorique, même si elle est risquée.
Exemple — Falaise (Cliff Walking)
Q-Learning emprunte le raccourci le long de la falaise (max reward théorique), même si une chute est possible pendant l'exploration. SARSA évite la falaise.
SARSA
SARSA
State-Action-Reward-State-Action · On-Policy · Model-Free · TD Learning.
On-PolicyModel-FreeValue-BasedTD Learning
Formule de mise à jour
Q(s,a) ← Q(s,a) + α × ( R + γ · Q(s', a') − Q(s,a) )
Différence clé vs Q-Learning : Q-Learning utilise max Q(s',a') — SARSA utilise Q(s',a' réellement choisie.
On-Policy vs Off-Policy
Q-Learning — Off-Policy
Apprend la politique optimale théorique quelle que soit l'exploration.
SARSA — On-Policy
Apprend la politique qu'il exécute réellement. Conservateur, évite les zones dangereuses.

PPO
PPO — Proximal Policy Optimization
On-Policy · Policy-Based · Model-Free. Développé par OpenAI pour stabiliser l'entraînement.
On-PolicyModel-FreePolicy-BasedOpenAIActor-Critic
Fonction de perte clippée
LCLIP(θ) = E[ min( rt(θ) · At , clip(rt, 1−ε, 1+ε) · At ) ]
r_t(θ)
Ratio nouvelle / ancienne politique
A_t
Avantage = Q(s,a) − V(s)
Clip [1−ε, 1+ε]
Bloque les mises à jour excessives
ε ≈ 0.2
Intervalle de tolérance [0.8, 1.2]
Architecture Acteur-Critique
Acteur (π) — hérite de Vanilla PG
Choisit directement l'action à prendre. C'est lui qu'on améliore sans le "casser" (clip).
Critique (V) — hérite du Q-Learning
Évalue la valeur V(s) de l'état actuel. Sert de juge pour calculer l'avantage A_t.
Pourquoi PPO ?
Problème Vanilla PG
Mises à jour trop grandes → Catastrophic Forgetting.
Solution PPO
Clipping contraint r_t ∈ [0.8, 1.2]. Hors plage → gradient ignoré. Apprentissage stable.
Avantage actions continues
Contrairement à Q-Learning, PPO gère les espaces d'actions infinis.
Dyna-Q
Dyna-Q
Off-Policy · Model-Based · Value-Based. Q-Learning + modèle interne de simulation.
Off-PolicyModel-BasedValue-BasedTD + Simulation
Deux sources d'apprentissage
Direct RL: Q(S,A) ← Q(S,A) + α[R + γ·max Q(S',A') − Q(S,A)]
Modèle: Model(S,A) = (R, S') ← stocker en mémoire
Planif.: N × Q-update sur expériences simulées via modèle
Avantage clé : Q-Learning pur est sample inefficient. Dyna-Q simule N expériences après chaque pas → convergence beaucoup plus rapide.
Boucle algorithmique — 4 étapes
1
Action réelle
Observer S, choisir A (ε-greedy), observer R et S'.
2
Direct RL
Appliquer Q-Learning classique sur (S,A,R,S').
3
Mise à jour modèle
Enregistrer Model(S,A) = (R, S') en mémoire interne.
4
Planification × N
Piocher (S,A) connus, interroger modèle, faire N MàJ Q simulées.
